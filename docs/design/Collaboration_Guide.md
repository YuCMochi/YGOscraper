# YGO Scraper 重構計劃 - 協作指南
## Mochi + Claude 合作工作流

---

## 📋 協作模式

### 模式 1: 同步討論和快速反饋
**適合**: 設計決策、技術討論、卡住時求助

```
流程:
Mochi: 提出問題 / 展示設計草稿
    ↓
Claude: 立即分析和提供建議
    ↓
Mochi: 快速迭代
    ↓
Claude: 驗證改進
```

**何時用**: 
- 設計系統色彩選擇
- 元件變體設計
- 代碼架構討論
- 性能優化

### 模式 2: 檢查點評估
**適合**: 階段完成後的全面檢查

```
流程:
Mochi: 完成一個階段的交付物
    ↓
提交交付物清單檢查
    ↓
Claude: 逐項檢查和評價
    ↓
提供改進建議和下一步指引
```

**何時用**:
- 線框圖完成
- 設計系統定稿
- 前端架構初步完成
- 測試報告完成

### 模式 3: 代碼審查和優化
**適合**: 開發期間的技術指導

```
流程:
Mochi: 完成一個功能模組或元件
    ↓
分享代碼和簡短說明
    ↓
Claude: 
  • 檢查代碼品質
  • 提供優化建議
  • 指出潛在問題
    ↓
Mochi: 迭代改進
```

**何時用**:
- 完成新元件
- 新頁面實現
- Hook 邏輯完成
- 服務層實現

```

```

**方式 2: 分享 GitHub 連結 (適合完整代碼)**
```
Mochi: 「已推送到 feat/button 分支：
https://github.com/YuCMochi/YGOscraper/blob/feat/button/src/components/Button.tsx
請幫我審查一下」
```

**方式 3: 共享檔案截圖 (適合設計審查)**
```
Mochi: 「Figma 首頁設計已完成，截圖如下...
有什麼改進的地方嗎？」
```

### 詢問的最佳方式

**詢問前做的準備**:
1. 清楚描述你遇到的問題
2. 提供相關的上下文
3. 列出已嘗試的解決方案
4. 明確你想要的幫助

**好的詢問範例**:
```
「我在做 Input 元件，需要支援以下變體：
- Search (左邊有放大鏡)
- Number (左右有 +/- 按鈕)
- 錯誤狀態 (邊框變紅)

我看了設計系統，但不確定：
1. Focus 狀態的外環光應該多大？
2. Error 狀態下的錯誤提示應該怎樣排版？

能幫我審查一下我的實現嗎？[代碼]」
```

**不太好的詢問範例**:
```
「我的代碼有問題，能幫我修嗎？」
```

## ⚡ 快速參考

### 常見問題速查

**Q: 我卡住了，不知道如何開始？**
A: 按照計劃書的優先順序，從簡單的開始。如果不知道方向，寫下具體困惑點，我幫你分解。

**Q: 我的進度比預期慢怎麼辦？**
A: 完全正常！調整預期，集中在關鍵交付物上。我可以幫你簡化非關鍵功能。

**Q: 我想改變某個設計決策怎麼辦？**
A: 隨時可以改！設計是迭代的。列出改動原因和新方案，我幫你評估。

**Q: 我想要額外的挑戰任務怎麼辦？**
A: 太好了！我可以提供進階課題，例如：
  - 虛擬化長列表 (performance)
  - 黑暗模式完整實現
  - 國際化 (i18n) 支援
  - PWA 功能
